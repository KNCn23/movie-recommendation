A content-based movie recommender over **9,500+ films**. Pick a movie and it
instantly returns the most similar titles — by plot, genre and quality — with
posters, ratings and a bilingual (EN/TR) interface.

**▶ Try it live: https://kncn23.github.io/movie-recommendation/**

## How it works

GitHub Pages serves only static files, so there is **no server at runtime**. The
expensive part — computing similarities across the whole catalog — runs **once,
offline**, and the result is baked into a single JSON the browser reads:

```
data/movies.csv ──▶ engine/recommender.py ──▶ engine/build_data.py ──▶ docs/data/movies.json ──▶ static site
   (TMDB, CC0)         TF-IDF + cosine            export top-N             (~5 MB, gzip ~2 MB)      (GitHub Pages)
```

### The recommendation model

For each film, similarity to every other film combines three signals
([`engine/recommender.py`](engine/recommender.py)):

| Signal | How | Weight |
|---|---|---|
| **Plot** | TF-IDF (1–2 grams) over the overview, cosine similarity | `1.0` |
| **Genre** | Multi-hot genre vector, cosine similarity | `0.9` |
| **Quality** | Candidate's rating ÷ 10, as a tie-breaker | `0.05` |

Genres are kept as a *separate* multi-hot signal rather than mixed into the text:
as plain words their high document-frequency makes TF-IDF ignore them, yet
"shares the same genres" is exactly what makes two films feel alike. The small
quality term breaks ties toward better-rated films, so look-alikes aren't
obscure. The top-12 neighbours per film are computed in batches (no full
N×N matrix) and exported.

## Features

- 🔎 **Instant search** with autocomplete (title + mini-poster + rating)
- 🎯 **"More like this"** — 12 content-based recommendations per film
- 🗂️ **Browse by genre** (19 genres) and **Popular / Top-rated** rows
- 🖼️ Posters straight from the TMDB image CDN (no API key needed)
- 🌐 **Bilingual UI** — EN/TR toggle, remembered across visits
- 🔗 Shareable deep links (`#/movie/…`, `#/genre/…`) with working back button
- 📱 Responsive, dark, Netflix-style UI — zero dependencies, zero build step

## Project structure

```
engine/
  recommender.py     # content-based engine (also a CLI)
  build_data.py      # exports docs/data/movies.json
  requirements.txt
data/
  movies.csv         # source dataset (TMDB, CC0)
docs/                # ← the live GitHub Pages site
  index.html
  css/styles.css
  js/app.js  js/i18n.js
  data/movies.json   # generated by build_data.py
```

## Run it locally

### Get recommendations from the CLI

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r engine/requirements.txt
python engine/recommender.py "Interstellar"
python engine/recommender.py "Toy Story" --n 15
```

### Rebuild the site data

```bash
python engine/build_data.py        # writes docs/data/movies.json
```

### Preview the site

```bash
python -m http.server -d docs 8000   # then open http://localhost:8000
```

## Dataset

Movie metadata and poster URLs come from the
[Pablinho/movies-dataset](https://huggingface.co/datasets/Pablinho/movies-dataset)
(sourced from **TMDB**), released under **CC0 1.0** (public domain). Posters are
loaded directly from `image.tmdb.org`, which requires no API key.
